Plano de Execução (Desenvolvimento)

Roadmap passo a passo para construção do MVP do Licitus Bot

1 Ambiente & Infraestrutura Base

  • 1.1. Inicialização do repositório Git e criação dos documentos base (commit.html, readme.md).
  • 1.2. Configuração do ambiente Python virtual (venv) e dependências (Selenium/Playwright, Requests, Flask/FastAPI).
  • 1.3. Estruturação de pastas do projeto (src/ bot/ web/ docs/ tests/).

2 Autenticação & Exploração do Portal (Gov.br)

  • 2.1. Desenvolvimento do script de login automatizado via Gov.br usando credenciais seguras.
  • 2.2. Mapeamento do DOM/APIs internas do site Compras.gov.br na página de pregões eletrônicos.
  • 2.3. Criação da função de navegação até a sala de disputa de um pregão específico.

3 Core Automático (O Robô de Lances)

  • 3.1. Loop de leitura em tempo real do "Menor Lance Atual".
  • 3.2. Lógica de cálculo: Se Menor Lance > Valor Mínimo, Novo Lance = Menor Lance - Decremento.
  • 3.3. Envio do lance via simulação de clique ou requisição POST HTTP direta.
  • 3.4. Mecanismo de Rate Limiting (esperar 3s-5s entre ações para não ser bloqueado).

4 Interface Gráfica Local (Desktop/Web MVP)

  • 4.1. Desenvolvimento de interface simples para inserir UASG, Número do Pregão e Item.
  • 4.2. Campos para "Valor Mínimo (Proteção)" e "Valor do Decremento".
  • 4.3. Botões de Iniciar / Pausar roteiro do bot.
  • 4.4. Terminal visual (log) na tela mostrando o status (Lendo lances... Lance enviado...).

5 Testes Homologação & Deploy

  • 5.1. Rodar bot em sala de treinamento/homologação do Compras.gov.br (se disponível) ou pregão simulado seguro.
  • 5.2. Criação do executável (.exe) ou script de inicialização do servidor local (.bat).
  • 5.3. Documentação técnica de uso para o usuário final.
